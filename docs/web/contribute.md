## Contribution Flow

1. Open an issue or propose a PR scope.
2. Make focused edits to instructions/docs/tools.
3. Validate locally (build, lint, checks relevant to your change).
4. Submit PR with rationale and expected behavioral impact.

## Prompt Artifact Review (Required For Prompt Changes)

Use the `prompt-authoring` skill for prompt-related updates.

1. Create/update a concise prompt brief (goal, non-goals, constraints, HITL points).
2. Generate or refactor the target prompt artifact.
3. Run validation against a checklist/rubric (structure, quality, correctness, governance).
4. Attach validation evidence in the PR description.

## Minimal Quality Bar

- Keep scope narrow and explicit.
- Avoid duplicate rules and ambiguous wording.
- Preserve safety, privacy, and approval checkpoints.

<div class="note">If you touched architecture decisions, update context/architecture docs and implementation status notes in the same change set.</div>
